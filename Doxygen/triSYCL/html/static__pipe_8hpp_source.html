30/** A SYCL static-scoped pipe equivalent to an OpenCL program-scoped
31 pipe
32
33 Implement a FIFO-style object that can be used through accessors
34 to send some objects T from the input to the output.
35
36 Compared to a normal pipe, a static_pipe takes a constexpr size
37 and is expected to be declared in a compile-unit static context so
38 the compiler can generate everything at compile time.
39
40 This is useful to generate a fixed and optimized hardware
41 implementation on FPGA for example, where the interconnection
42 graph can be also inferred at compile time.
43
44 It is not directly mapped to the OpenCL program-scoped pipe
45 because in SYCL there is not this concept of separated
46 program. But the SYCL device compiler is expected to generate some
47 OpenCL program(s) with program-scoped pipes when a SYCL
48 static-scoped pipe is used. These details are implementation
49 defined.
50*/
51template <typename T, std::size_t Capacity>
52class static_pipe
53 /* Use the underlying pipe implementation that can be shared in
54 the SYCL model */
55 : public detail::shared_ptr_implementation<static_pipe<T, Capacity>,
56 detail::sycl_2_2::pipe<T>>,
57 detail::debug<static_pipe<T, Capacity>> {

60 using implementation_t = typename static_pipe::shared_ptr_implementation;

63 using implementation_t::implementation;

66 friend implementation_t;
